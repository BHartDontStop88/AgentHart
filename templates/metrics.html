What do these metrics mean?
Tokens/Run — Total prompt + response tokens consumed per LLM call. Lower = more efficient prompt engineering.
TPS (Tokens/sec) — Gemma4's generation speed. Higher = model cached in RAM. Drop indicates cold model load.
Model Load ms — Time to load model weights. Near-zero = model already in memory (warm cache). 2-3s = cold load.
Ctx Window % — Percentage of Gemma4's context window used per call. Over 80% risks truncation.
Outputs — Useful items produced: tasks created, reports written, summaries saved, alerts sent.
Token Economy bar — Blue = prompt tokens sent in. Purple = response tokens generated. Mostly blue = verbose prompts.
Success Rate — % of runs that completed without error. Below 80% = investigate agent logs.
